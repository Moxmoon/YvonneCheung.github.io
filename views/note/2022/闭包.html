介绍闭包及其应用场景，作用域的概念。闭包会造成内存泄漏吗？

# 闭包的概念

:::info 指那些能够访问自由变量的函数。 自由变量指函数中使用的，既不是函数参数也不是函数局部变量的变量。

  1. 从理论角度：所有的函数都是闭包
  2. 从实践角度：
    1. 即使创建它的上下文已经销毁，它仍然存在（比如：内部函数，从父函数返回）
    2. 代码中引用了自由变量 :::

# 闭包的应用场景

# 柯里化函数

:::info 避免频繁调用具有相同参数的函数，同时又可以轻松复用 封装一个高阶函数 :::

function getArea(width, height) {
    return width * height
}

const area1 = getArea(10, 20)
const area2 = getArea(10, 30)
const area3 = getArea(10, 40)

function getArea(width) {
    return height => {
        width * height
    }
}

const getTenWidthArea = getArea(10)
const area4 = getTenWidthArea(20)
const area5 = getTenWidthArea(30)

# 实现私有方法/变量

:::info 其实就是模块的方式，现代化的打包最终其实就是每个模块的代码相互独立 :::

# 匿名自执行函数

var funOne = (function () {
    var num = 0;
    return function () {
        num++;
        return num;
    }
})()

console.log(funOne()) // 1
console.log(funOne()) // 2
console.log(funOne()) // 3

# 缓存一些结果

:::info 在外部函数创建一个数组，闭包函数可以更改/获取数组的值。 延续局部变量的寿命 :::

function funParent() {
    let memo = [];
    function funTwo(i) {
        memo.push(i);
        console.log(memo.join(','))
    }

    return funTwo
}

const fn = funParent()

fn(1) // 1
fn(2) // 1,2
fn(3) // 1,2,3

# 总结

  • 创建私有变量
  • 延长变量的生命周期

# 面试题

function fn1(x) {
    return x + 1
}
function fn2(x) {
    return x + 2
}
function fn3(x) {
    return x + 3
}
function fn4(x) {
    return x + 4
}

function compose() {
    const argFnList = [...arguments]

    return (num) => {
        return argFnList.reduce((pre, cur) => cur(pre), num)
    }
}

const a = compose(fn1, fn2, fn3, fn4)
console.log(a(5))

function currying(fn, ...args) {

    const originFnArgumentLength = fn.length
    let allArgs = [...args]

    const resFn = (...newArgs) => {
        allArgs = [...allArgs, ...newArgs]

        if (allArgs.length === originFnArgumentLength) {
            return fn(...allArgs)
        } else {
            return resFn
        }
    }
    return resFn
}

const add = (a, b, c) => a + b + c

const a1 = currying(add, 1)
const a2 = a1(2);
console.log(a2(3)) // 肯定是返回一个函数

# 闭包会造成内存泄漏吗？

:::info 不会，内存泄漏是指你用不到（访问不到）的变量，依然占据着内存空间，不能被再次利用起来。把一些变量放在闭包环境中，和全局中影响是一样的，并不能说成是闭包泄漏，如果需要回收这些变量，可以手动设为 null。 有关系的地方是，使用闭包的同时比较容易形成循环引用，如果闭包的作用域链中保存着一些DOM节点，就有可能造成内存泄漏，并非闭包和JS的问题，而是IE浏览器中，由于BOM和DOM中的对象是使用C++以COM的对象方式实现的，而COM对象的垃圾收集机制采用的是引用技术策略，在基于引用技术策略的垃圾回收机制中，如果两个对象之间形成了循环引用，那么这两个对象都无法被回收，但循环引用造成的内存泄漏在本质上也不是闭包造成的。 同样，如果要解决循环引用带来的内存泄漏问题，只需要把变量设为 null 即可。设为 null 意味着切断变量与他此前引用值之前的连接。当垃圾收集器下次运行时，就会删除这些值并回收它们占用的内存。 :::

# 块级作用域的特点

  • 声明变量不会被提升到代码块顶部
  • 禁止重复声明
  • 变量只在当前块有效
for (var i = 0; i < 5; i++) {
    setTimeout(function () {
        console.log(i) // 10 10 10 10 10
    })
}

for (let i = 0; i < 5; i++) {
    setTimeout(function () {
        console.log(i) // 0 1 2 3 4
    })
}

function v() {
    var a = 6;
    function a() { }
    console.log(a) // 6
}

function v() {
    var a;
    function a() { } // 函数作用域的提升优先级更高，因为它在后面，肯定会覆盖掉变量提升的var

    console.log(a) // [Function: a]
}
